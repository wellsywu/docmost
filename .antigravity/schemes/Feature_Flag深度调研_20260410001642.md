## 二、关键代码节点详解

### 2.1 后端 — `resolveFeatures()` （**核心拦截点**）

```typescript
// license-check.service.ts 第 65-77 行
resolveFeatures(licenseKey: string, plan: string): string[] {
    if (this.environmentService.isCloud()) {
      try {
        const { getFeaturesForCloudPlan } = require('../../ee/licence/feature-registry');
        return [...getFeaturesForCloudPlan(plan)];
      } catch {
        return [];  // ← Cloud 环境下 EE 模块缺失
      }
    }
    return this.getFeatures(licenseKey);
    // ↑ 自部署环境走这里，内部也是 require EE 模块
    // 无 EE 模块时返回 []
}
```

> [!CAUTION]
> **这是所有 Feature Flag 的总开关。** 无 EE 模块 → `getFeatures()` catch 返回 `[]` → 前端所有受 Feature 控制的功能全部禁用。

### 2.2 前端 — `entitlements` 端点返回值

后端 `POST /workspace/entitlements` 返回的数据结构：

```typescript
// 有 EE License 时
{ cloud: false, tier: "standard", features: ["api:keys", "sso:custom", "mfa", ...] }

// 无 EE License 时（当前社区版）
{ cloud: false, tier: "free", features: [] }
```

### 2.3 前端 — 侧边栏入口控制

```typescript
// settings-sidebar.tsx 第 148-162 行
const hasFeature = (f: string) =>
    entitlements?.features?.includes(f) ?? false;

const isItemDisabled = (item: DataItem) => {
    if (!item.feature) return false;    // 无 feature 属性 → 不禁用
    return !hasFeature(item.feature);   // 有 feature 但不在列表中 → 禁用
};
```

API Key 相关菜单项定义：
| 菜单项 | 路径 | feature | role |
|--------|------|---------|------|
| "API keys" | `/settings/account/api-keys` | `api:keys` | 无 |
| "API management" | `/settings/api-keys` | `api:keys` | admin |

> [!IMPORTANT]
> 菜单项仅被 **禁用**（灰色不可点击），**不是隐藏**。禁用时显示 Tooltip 提示 "Available with a paid license"。

### 2.4 前端 — API Key 页面内部检查

| 组件 | Feature 检查 | 影响 |
|------|-------------|------|
| `user-api-keys.tsx` | **无** Feature 检查 | 页面直接可用 |
| `workspace-api-keys.tsx` | **无** Feature 检查 | 页面直接可用 |
| `restrict-api-to-admins.tsx` | `useHasFeature(Feature.API_KEYS)` | 无 Feature 时开关被禁用 |

**结论**：即使通过 URL 直接访问 `/settings/account/api-keys`，页面组件本身不会拒绝渲染。但缺少后端 API 端点会导致请求失败。

## 三、解锁方案对比

### 方案 A：修改 `resolveFeatures()`（推荐）

**改动文件**：[license-check.service.ts](file:///d:/dev_workspace/docmost/apps/server/src/integrations/environment/license-check.service.ts)

```diff
 resolveFeatures(licenseKey: string, plan: string): string[] {
     if (this.environmentService.isCloud()) {
       try {
         const { getFeaturesForCloudPlan } = require('../../ee/licence/feature-registry');
         return [...getFeaturesForCloudPlan(plan)];
       } catch {
         return [];
       }
     }
-    return this.getFeatures(licenseKey);
+    const features = this.getFeatures(licenseKey);
+    // 社区版默认启用 API Key 功能
+    if (!features.includes('api:keys')) {
+      features.push('api:keys');
+    }
+    return features;
 }
```

**优点**：改动最小（3 行），逻辑清晰
**缺点**：升级时若此方法签名变化需同步

### 方案 B：修改 `entitlements` 端点

**改动文件**：[workspace.controller.ts](file:///d:/dev_workspace/docmost/apps/server/src/core/workspace/controllers/workspace.controller.ts)

```diff
 @Post('entitlements')
 async getEntitlements(@AuthWorkspace() workspace: Workspace) {
-    return {
-      cloud: this.environmentService.isCloud(),
-      tier: this.licenseCheckService.resolveTier(licenseKey, plan),
-      features: this.licenseCheckService.resolveFeatures(licenseKey, plan),
-    };
+    const features = this.licenseCheckService.resolveFeatures(licenseKey, plan);
+    if (!features.includes('api:keys')) {
+      features.push('api:keys');
+    }
+    return {
+      cloud: this.environmentService.isCloud(),
+      tier: this.licenseCheckService.resolveTier(licenseKey, plan),

 }
```

**优点**：不修改底层 service，影响范围更小
**缺点**：controller 层塞入业务逻辑不太优雅

### 方案 C：前端绕过 Feature 检查

修改 `settings-sidebar.tsx` 跳过 `api:keys` 的 disabled 检查。

**缺点**：需要额外修改 `restrict-api-to-admins.tsx`，改动点分散，不推荐

## 四、还需要注意的 `hasFeature` 调用点

`useHasFeature(Feature.API_KEYS)` 在整个前端仅有 **1 处** 调用（除侧边栏外）：

| 文件 | 用途 | 影响 |
|------|------|------|
| `restrict-api-to-admins.tsx` 第 23 行 | 控制"限制 API Key 创建权限仅管理员"开关的启用/禁用 | 无 Feature 时开关被禁用但页面仍可渲染 |

**只要后端 entitlements 返回 `api:keys`，这个开关也会自动解锁。** 无需额外处理。

## 五、总结

| 维度 | 结论 |
|------|------|
| 拦截点 | `LicenseCheckService.resolveFeatures()` → 返回空数组 |
| 影响范围 | 侧边栏菜单禁用 + restrict-api-to-admins 开关禁用 |
| 页面级拦截 | **无**（页面组件不做 Feature 检查） |
| 推荐方案 | **方案 A**：修改 `resolveFeatures()` 添加 `api:keys`，改动 3 行 |
| 与后端 API Key 功能的关系 | 解锁 Feature Flag 是前端入口的前提条件，但不影响后端 API 端点的实现 |
